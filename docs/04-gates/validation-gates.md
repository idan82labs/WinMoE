# Validation Gate Map

Gate 0 -> repository and operating model
Gate 0A -> systems sanity and primitive-path selection
Gate 1 -> routing observability
Gate 2 -> calibrated timings
Gate 3 -> simulator viability
Gate 4 -> feasibility judgment
Gate 5 -> report lock

The project should not skip gates. If a gate is weakly defined, revise the gate file rather than drifting past it.
