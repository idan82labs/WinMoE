# Project Facts

## Current first-order facts
- The project is framed as a RAM-first caching feasibility study.
- Correct byte accounting and causal modeling invalidated earlier zero-cache optimism.
- Timing calibration must precede strong tok/s claims.
- DirectStorage is currently a later-stage optimization candidate, not a first-order savior.
- Systems-baseline work now precedes broad trace/simulator claims.
- The default expert-streaming baseline should use explicit reads and explicit staging unless evidence says otherwise.
- KV compression is a budget lever that may free memory for expert caching, not a direct substitute for expert-streaming analysis.
